# AI System Registration Form

> Complete this form before deploying any AI system.
> This is the artifact that answers: "Who approved this AI and what did they know when they approved it?"
> Maps to: GOVERN 1.1, GOVERN 1.2, MAP 1.1, MAP 2.3
> Corresponding 800-53: PL-2, RA-3, PM-10

---

## Instructions

Fill out every field. "N/A" requires a written justification. Vague answers like "used for security" are not acceptable — be specific.

After completion, this form is signed by the ISSO and retained as a system authorization artifact. Update it when the system changes materially (new model version, new data source, expanded scope).

## Completed Example — BERU v1.0

| Field | Value |
|---|---|
| System Name | BERU |
| System Identifier | BERU-v1.0 |
| Version | v1.0 |
| System Owner | J (jimjrxieb) |
| ISSO | jimjrxieb |
| Technical Contact | jimjrxieb (MLOps engineer) |

**What it does:** Reads security scanner output from GP-Copilot's 5 C's toolchain (Trivy, kube-bench, Prowler, Kubescape, Falco). Maps findings to NIST 800-53 Rev 5 controls and NIST AI RMF subcategories. Produces structured 9-field findings, POA&M draft items, SSP narrative guidance, and CISO briefing summaries.

**What it does NOT do:** Approve risk acceptances above C-rank. Execute remediation commands. Access production systems directly. Make B/S-rank decisions autonomously.

**Risk Tier:** Minimal Risk — advisory output only. All B/S-rank findings routed to human review queue before finalization. Human confirms before any POA&M item is filed with AO.
