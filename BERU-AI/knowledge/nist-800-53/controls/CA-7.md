id: CA-7
name: Continuous Monitoring
---

question: "Is there an ongoing program that continuously evaluates security control effectiveness — not just at assessment time?"

description: >
  The organization develops a continuous monitoring strategy and implements a continuous
  monitoring program that includes establishing defined metrics; monitoring controls on a
  defined frequency; assessing security and privacy controls at a defined frequency;
  ongoing status reporting; correlating and analyzing security information generated by
  assessments and monitoring; and responding to the results. Continuous monitoring is
  the bridge between the periodic control assessment (CA-2) and the operational state of
  the system. Controls that pass assessment can degrade between assessments through
  configuration drift, personnel turnover, or evolving threats. Continuous monitoring
  detects that degradation before it becomes a breach.

enhancements:
  - id: CA-7(1)
    name: Independent Assessment
    description: >
      The organization employs assessors with defined independence to monitor controls
      on an ongoing basis. Continuous monitoring is not self-monitoring — independent
      verification of control effectiveness, even if lightweight and automated, provides
      an objective signal that is not subject to the bias of the team operating the controls.
  - id: CA-7(4)
    name: Risk Monitoring
    description: >
      The organization ensures risk monitoring is an integral part of the continuous
      monitoring program. Monitoring covers not only control status but the risk environment:
      threat intelligence feeds, vulnerability publications, changes to the threat landscape
      relevant to the system. Risk posture can change without any change to the system itself.

evidence:
  what_to_look_for:
    - Continuous monitoring strategy document defining metrics, frequencies, responsible parties, and reporting
    - Automated control status dashboard — live view of control health across the system
    - Scheduled reporting cadence (monthly or quarterly continuous monitoring report to ISSO/AO)
    - Evidence that monitoring results feed into the POA&M and risk register — findings are acted on
    - Metrics tracked over time — trend data showing whether the security posture is improving or degrading
    - Integration of threat intelligence into monitoring — controls re-evaluated when relevant new TTPs emerge
  ask_for:
    - "Show me your continuous monitoring strategy — what controls are monitored at what frequency, and who is responsible?"
    - "Show me your most recent continuous monitoring report — what was the control status, and what findings were escalated?"
    - "Show me how a drift from approved configuration detected by continuous monitoring gets escalated and remediated."
    - "Show me your metrics trend over the last quarter — is your security posture improving, degrading, or stable?"
  tools:
    generic:
      - Prometheus + Grafana (metrics-based continuous monitoring with alerting thresholds)
      - ArgoCD (continuous GitOps state monitoring — drift detection against desired state)
      - Falco (continuous runtime behavioral monitoring)
      - kube-bench (scheduled CIS benchmark checks — continuous configuration compliance)
    aws:
      - AWS Security Hub (continuous aggregated finding dashboard — control status at a glance)
      - AWS Config (continuous configuration compliance monitoring — rule evaluation on change)
      - Amazon GuardDuty (continuous threat detection — always-on, no configuration required)
      - AWS Audit Manager (continuous evidence collection and assessment dashboard)
    microsoft:
      - Microsoft Defender for Cloud (continuous security score — real-time control health)
      - Microsoft Sentinel (continuous SIEM monitoring with scheduled detection rules)
      - Microsoft Purview Compliance Manager (continuous compliance score with action tracking)
      - Azure Policy (continuous compliance evaluation — non-compliant resources flagged in real time)

failure_to_implement:
  - No continuous monitoring strategy — the organization relies entirely on annual assessments to know control status.
  - Monitoring tools deployed but no reporting cadence — data is collected but never synthesized into a status view.
  - Continuous monitoring findings not linked to the POA&M — gaps are detected and then ignored.
  - Security posture is measured at a point in time but no trend data exists — cannot demonstrate improvement.
  - Annual assessment passes but controls degraded between assessments — continuous monitoring would have caught the drift.
