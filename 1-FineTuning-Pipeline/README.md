# 1-FineTuning-Pipeline — Training Engine

7-step LoRA fine-tuning pipeline for BERU, JADE, and Katie running on local GPU (RTX 5080). Data flows forward through numbered directories. Files archive after each step. Includes a closed feedback loop from eval gaps back to training.

**Primary model:** BERU (GRC analyst, 3B). JADE (DevSecOps, 8B) and Katie (K8s ops, 3B) use the same pipeline with different configs.

---

## Pipeline Flow

```text
04-processed/        ← Curated/archived data re-ingested here
01-raw-data-lake/    ← Drop raw data here (gitignored)
       │
   etl_pipeline.py   STEP 1: Extract, Transform, Load
       │
02-ETL-data/         Cleaned, deduplicated, labeled JSONL (gitignored)
       │
   chunk_data.py     STEP 2: Split into 5k-10k chunks + 5% eval holdout
       │
03-chunked-untrained/ Chunks ready for training (gitignored)
       │
   train_v11.py      STEP 3: LoRA fine-tune on RTX 5080 (~45 min/chunk for 3B)
       │
   merge_model.py    STEP 4: Merge LoRA adapter into base model
       │
   convert_gguf.py   STEP 5: Convert to GGUF for Ollama
       │
   eval_bridge.py    STEP 6: Benchmark against 4-eval-clarify/ suites
       │
   feedback_loop.py  STEP 7: Identify weak categories → generate gap data
       │
04-processed/        Loop closes — gap data staged for next ETL cycle
```

## Directory Structure

```text
1-FineTuning-Pipeline/
├── 01-raw-data-lake/       ← Raw input (gitignored — drop JSONL here)
├── 02-ETL-data/            ← Cleaned JSONL output from etl_pipeline.py (gitignored)
├── 03-chunked-untrained/   ← 5k-10k chunks ready for training (gitignored)
├── 04-processed/           ← Archive: curated data + feedback loop output
├── 06-eval-holdout/        ← 5% holdout set held back from training (gitignored)
├── rank-training-data/

│   └── rank_classifier.metrics.json ← classifier performance

├── etl_pipeline.py         ← Step 1
├── chunk_data.py           ← Step 2
├── train_v11.py            ← Step 3 (current — Unsloth + LoRA)
├── merge_model.py          ← Step 4
├── convert_gguf.py         ← Step 5
├── eval_bridge.py          ← Step 6
├── feedback_loop.py        ← Step 7
├── config.yaml             ← Default training config (LoRA r=64/alpha=128, 4-bit)

```

## Quick Start

```bash
cd GP-MODEL-OPS/1-FineTuning-Pipeline

# Step 1: ETL — normalize and deduplicate
python3 etl_pipeline.py --dry-run   # preview first
python3 etl_pipeline.py

# Step 2: Chunk
python3 chunk_data.py --shuffle

# Step 3: Train (auto-resumes from last checkpoint)
python3 train_v11.py --status       # check where you are
python3 train_v11.py                # train next chunk
python3 train_v11.py --loop         # train all remaining chunks

# Step 4-5: Merge + GGUF
python3 merge_model.py
python3 convert_gguf.py

# Deploy to Ollama
ollama create beru:v1.7 -f 3-model-registry/beru/v1.7/Modelfile

# Step 6-7: Eval + feedback
python3 eval_bridge.py --latest
python3 feedback_loop.py --latest --export
```

## Training Config (`config.yaml`)

| Parameter | Value | Notes |
|-----------|-------|-------|
| Base model (3B) | `unsloth/Llama-3.2-3B-Instruct` | BERU, Katie |
| Base model (8B) | `unsloth/Llama-3.1-8B-Instruct` | JADE |
| LoRA r | 64 | |
| LoRA alpha | 128 | |
| Quantization | 4-bit | Unsloth |
| Max seq length | 8192 | 4096 was truncating eval prompts — fixed in exp-007 |
| Epochs/chunk | 2 | Cosine LR scheduler |
| Batch size | 4 | Gradient accumulation 8 → effective 32 |
| GPU | RTX 5080 | ~45 min/chunk for 3B, ~2h for 8B |

BERU-specific overrides in `config_beru.yaml` (different corpus paths, LoRA dims).

## ETL Input Formats

`etl_pipeline.py` reads from both `04-processed/` and `01-raw-data-lake/` and normalizes to ChatML:

| Format | Structure |
|--------|-----------|
| ChatML | `{"messages": [{"role": "user", ...}, {"role": "assistant", ...}]}` |
| Alpaca | `{"instruction": "...", "input": "...", "output": "..."}` |
| Q&A | `{"question": "...", "answer": "..."}` |

Alpaca is normalized to ChatML on ingest. All output is ChatML only.

---

## What's Gitignored

The large data directories are gitignored — regenerate from source:

```bash
python3 etl_pipeline.py        # regenerates 02-ETL-data/
python3 chunk_data.py          # regenerates 03-chunked-untrained/
```

Model weights, GGUFs, and checkpoints are also gitignored. They live in `3-model-registry/` locally and GP-S3 for backups.

The `rank-training-data/rank_classifier.joblib` is tracked — it's the trained sklearn classifier, not a large binary.
